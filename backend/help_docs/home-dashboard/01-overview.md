# Home Dashboard overview

The **Home Dashboard** is your customizable **home page**: a layout of **widgets** you arrange on a grid. Use it for quick links, notes, RSS headlines, org agendas, folders, pinned documents, and image slideshows.

---

## What it is

- Each **dashboard** has a saved **layout** (list of widgets and their positions).
- You can have multiple dashboards; one may be marked **default** and is used when you open **Home** without a specific id.
- **Edit mode** lets you change widgets and the grid; leaving edit mode saves when you confirm.

---

## Widget types

Available widget types include:

| Type | Purpose |
|------|--------|
| **Navigation links** | Quick links to app areas or external URLs. |
| **Markdown note** | Freeform markdown content in a card. |
| **RSS headlines** | Headlines from configured RSS feeds. |
| **Org agenda** | Org-mode style agenda block. |
| **Folder shortcuts** | Jump links for chosen folders. |
| **Pinned documents** | Shortcuts to specific documents. |
| **Folder images** | Slideshow or gallery of images from a folder. |

---

## Adding and editing widgets

- Use **add** actions (e.g. from the chrome/menu) to insert a new widget; each type gets a **sensible default size** on the 12-column grid.
- **Drag** widgets to **rearrange**; **resize** handles change width and height in grid units.
- Open a widget’s **editor** to set title, content, feed ids, folder ids, or other type-specific options.
- New widgets without positions are **auto-placed** in a simple two-column flow when needed.

---

## Responsive layout

The dashboard uses a **responsive grid** with breakpoints (large / medium / small / extra-small). Column counts adjust (for example, **12** columns on wide screens and fewer on narrow ones) so the same layout **reflows** instead of overflowing. **Stack**-style layout modes may strip explicit grid coordinates and order widgets by their logical position for narrow views.

---

## Summary

The Home Dashboard combines **markdown**, **navigation**, **RSS**, **org**, **folders**, **documents**, and **images** in one place. Edit mode controls **what** appears and **where**; the grid **adapts** to screen size.
